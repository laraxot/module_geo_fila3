Form Search Address Categories

By

<h1 id="form-search-address-categories">Form Search Address Categories</h1> <p>Richiama a sua volta un componente livewire (quindi in "real-time") per cercare un indirizzo usando le API di Google Maps</p> <p>Nome Componente:</p> <pre><code class="language-php">x-form-search-address-categories </code></pre> <p>Non richiede parametri</p> <p>E' necessario utilizzare l'API Key di Google Maps, che va impostata in:</p> <pre><code class="language-php">config('services.google.maps_key') </code></pre> <p>cioè nel file:</p> <pre><code class="language-console">config/services.php </code></pre> <p>Esempio:</p> <pre><code class="language-php">&lt;x-form-search-address-categories&gt; &lt;/x-form-search-address-categories&gt; </code></pre> <p>Per altre informazioni leggere documentazione sulle <a href="https://developers.google.com/maps/documentation/javascript/overview">API di Google Maps</a>.</p>

Aggiornamenti

By

<h1 id="aggiornamenti">Aggiornamenti</h1> <p>Cms onsiste in un pacchetto PHP.</p> <p>Quindi per aggiornare basta dare il comando in console:</p> <p>composer update laraxot/module_geo</p> <p>Assicurarsi di cancellare la cache delle viste di Laravel dopo l'aggiornamento:</p> <pre><code class="language-console">php artisan view:clear </code></pre> <p>Infine, se si sono pubblicati il file di configurazione o i modelli Blade, assicurarsi che le personalizzazioni siano aggiornate con i valori predefiniti.</p> <h3>Verifica dei pacchetti installati</h3> <p>È possibile utilizzare il comando Artisan incorporato per visualizzare le versioni installate dei pacchetti:</p> <pre><code class="language-console">php artisan module_geo:show-versions </code></pre>
